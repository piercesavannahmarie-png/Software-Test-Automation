## How can I ensure that my code, program, or software is functional and secure?

I ensure my code is functional and secure by carefully following the software requirements and validating them through unit testing. In this course, I used JUnit tests to confirm that each class behaved as expected and handled both valid and invalid inputs correctly. Writing tests for boundary conditions and invalid data helped prevent errors and improved reliability. I also structured my classes to enforce constraints such as required fields, maximum character lengths, and unique identifiers, which helped strengthen the security and stability of the program.

## How do I interpret user needs and incorporate them into a program?

I interpret user needs by reviewing requirements carefully and translating them into clear program behaviors. For example, when developing the contact service, I used the provided requirements to determine what fields were required, what limits needed to be enforced, and what operations the service needed to support. I then implemented validation logic directly into the classes to ensure those requirements were always met. Unit testing helped confirm that the program behaved according to user expectations.

## How do I approach designing software?

I approach software design by first analyzing the requirements and breaking them into smaller components. I then create classes that represent those components and implement methods that support the required functionality. During development, I test each feature incrementally to make sure it works correctly before moving forward. This structured approach helps keep the code organized, readable, and easier to maintain. Using unit tests throughout development also helps identify issues early and improves overall software quality.
